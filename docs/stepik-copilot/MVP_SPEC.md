# MVP Specification

## Цель MVP

Доказать, что пользователю нужна "умная подсветка" текущего шага курса:
краткое объяснение, слабые места и выжимка по комментариям, встроенные прямо в
интерфейс Stepik.

## Сценарий использования

1. Пользователь устанавливает расширение.
2. Пользователь открывает шаг курса на Stepik.
3. Пользователь открывает сайдбар расширения.
4. Расширение собирает данные со страницы и отправляет их в AI-сервис.
5. Пользователь получает структурированный ответ:
   - краткое summary;
   - на что обратить внимание;
   - сигналы из комментариев;
   - чеклист перед выполнением задания.

## Что входит в MVP

### Обязательные функции

- поддержка страниц шагов Stepik;
- извлечение текста текущего шага из DOM;
- извлечение комментариев, видимых на странице;
- извлечение базового контекста: название шага, урока, модуля, курса;
- кнопка запуска анализа;
- сайдбар с результатом анализа;
- структурированный ответ AI по шаблону;
- защита от выдачи готового решения.

### Опционально для MVP+

- использование соседних шагов для контекста;
- облачная история анализов;
- персональные настройки глубины ответа;
- рекомендации по дополнительным источникам;
- сводка только по новым комментариям.

## Что не входит в MVP

- полноценная интеграция с Stepik API и OAuth как обязательное условие;
- мобильная версия;
- офлайн-режим;
- командный режим или shared notes;
- адаптация под другие LMS;
- голосовой интерфейс;
- режим автора курса.

## UX-состав ответа

AI-ответ должен быть коротким и предсказуемым по структуре:

- `О чем шаг` — 2–4 предложения;
- `На что обратить внимание` — 3–6 пунктов;
- `Что путает других` — 2–5 выводов по комментариям;
- `Проверь себя` — 3–5 пунктов перед отправкой ответа;
- `Нужен ли дополнительный контекст` — короткая пометка.

## Functional Requirements

### FE-1

Расширение должно отображать сайдбар на поддерживаемых страницах Stepik.

### FE-2

Пользователь должен иметь возможность запускать анализ вручную.

### FE-3

Расширение должно показывать состояние загрузки, ошибки и успешный результат.

### DATA-1

Content script должен извлекать из DOM:

- заголовок шага;
- текст шага;
- текст комментариев;
- базовые метаданные страницы, если они доступны визуально.

### AI-1

AI-сервис должен возвращать ответ в фиксированном JSON-формате.

### AI-2

AI-сервис должен следовать prompt-ограничениям:

- не решать задачу за пользователя;
- не подсказывать прямой ответ на тест;
- фокусироваться на понимании и рисках.

## Нефункциональные требования

- первый ответ желательно получать менее чем за 8 секунд;
- результат должен быть читаемым без прокрутки на типичном шаге;
- система не должна ломать верстку Stepik;
- пользователь должен явно понимать, какие данные уходят на анализ.

## Acceptance Criteria

- расширение ставится и запускается локально;
- на открытом шаге можно получить структурированный AI-ответ;
- для шага с комментариями система показывает отдельные выводы по комментариям;
- для шага без комментариев система корректно деградирует;
- система не выдает готовое решение в тестовом или кодовом шаге;
- при ошибке сети пользователь видит понятное сообщение и может повторить
  запрос.

# Roadmap

## Phase 0 — Discovery

Цель: зафиксировать MVP и убрать неопределенность до начала кода.

Задачи:

- описать product scope;
- выбрать стек расширения;
- понять минимально нужный payload;
- решить, нужен ли backend с первого дня;
- определить правила anti-cheating поведения.

Результат:

- согласованный пакет документации;
- список user stories;
- базовый prompt contract.

## Phase 1 — Local MVP

Цель: локально получить рабочий анализ шага Stepik.

Задачи:

- поднять каркас расширения;
- внедрить content script на страницах Stepik;
- извлечь текст шага и комментарии;
- добавить sidebar UI;
- поднять один backend endpoint;
- вывести summary, pitfalls и checklist.

Результат:

- локальная demo-версия, работающая на реальном шаге курса.

## Phase 2 — Usability Pass

Цель: довести продукт до первого понятного пользовательского опыта.

Задачи:

- улучшить формат ответа;
- добавить обработку пустых комментариев;
- ввести loading/error/empty states;
- сократить задержку ответа;
- добавить feedback-кнопки "полезно / не полезно".

Результат:

- более стабильный и читаемый UX.

## Phase 3 — Beta

Цель: начать тестирование на небольшой группе пользователей.

Задачи:

- ввести простую auth в собственном сервисе;
- добавить rate limits;
- логировать агрегированные продуктовые события;
- собрать первые qualitative interviews.

Результат:

- управляемая закрытая beta.

## Phase 4 — Expansion

Цель: перейти от базового copilot к более сильному продукту.
